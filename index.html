How does the per-project color cascade work in the sidebar? I want to reuse the pattern in my own app.

Claude generating a response

The trick is a single CSS custom property — --accent — set on each session row and read by every child that wants to tint. One write, everything downstream updates.

css 
/* Default accent for orphan sessions (pinned, recent) */.session { --accent: #8a8a92; }/* Dot and tree branch both read the same variable */.session__dot      { background: var(--accent); }.session.is-active .session__title {  color: color-mix(in srgb, var(--accent) 60%, #c4c4ca);}.project__body .session.is-active::before {  /* curved elbow border */  border-color: color-mix(in srgb, var(--accent) 45%, transparent);}

Then when you render sessions inside a project, just forward the project color as that one variable:

javascript 
function sessionRow(session, { accent = null } = {}) {  const row = el('div', { class: 'session' });  if (accent) row.style.setProperty('--accent', accent);  // ...build row content  return row;}// Inside projectRow(), forward the project color to each childproject.sessions.forEach(s =>  body.append(sessionRow(s, { accent: project.color })));

Change one color on the project object and every child row retints on next render — dots, branches, active-state tint, hover glow. No per-component prop drilling.
